Design the pre-login and rancher onboarding frontend for a livestock transport marketplace app called “Tropero”.

Important: this is a FRONTEND DEMO ONLY.
Do not design complex backend logic, real upload processing, OCR, or heavy technical states.
The goal is to create clean, realistic, investor-ready onboarding screens that look polished and easy to use.

Brand style:
- app name: Tropero
- aesthetic: mostly black and white
- accent colors inspired by the Paraguayan countryside: grass green, earth brown, subtle rural tones
- visual feel: modern, premium, simple, trustworthy, agricultural logistics
- logo: side profile silhouette of a Zebu / Cebu cattle animal
- should feel like a rural logistics startup, not a generic fintech

Target user for this flow:
Rancher / Ganadero

The rancher should feel:
- this is easy
- this saves time
- this is trustworthy
- this is clear and organized
- I can skip things and complete them later
- I am in control

Create a full frontend flow with separate screens.

SCREEN 1 — Welcome / Entry screen
Design a clean entry page before signup.
Include:
- Tropero logo
- headline
- short value proposition for ranchers
- two main buttons:
  - Iniciar sesión
  - Registrarte
- optional secondary link: Ver cómo funciona
This page should feel elegant, simple, and premium.

SCREEN 2 — Choose account type
Show account selection cards.
Main focus should be Rancher / Ganadero.
You may show other role options in smaller secondary style, but the rancher card should feel primary.
Roles:
- Ganadero
- Empresa
- Owner-operator
- Chofer de empresa

SCREEN 3 — Rancher signup step 1: basic information
Fields:
- First name
- Last name
- Phone number
- Email
- Ranch / farm name
- Region / department
Buttons:
- Continuar
- Omitir por ahora
- Guardar y seguir después

SCREEN 4 — Rancher signup step 2: ranch details
Fields:
- Ranch identifier or code
- RUC / tax ID
- Main cattle transport routes
- Approximate transport frequency
- Notes
Include a visible “Skip this step” option.
This screen should feel lightweight, not overwhelming.

SCREEN 5 — Rancher signup step 3: documents
Important:
This is only a visual demo frontend.
Do NOT design complex upload logic.
Use elegant upload cards / drag-and-drop boxes / placeholders for:
- RUC
- Ranch identification
- Other optional documents
Include:
- Upload later
- Skip this step
- Small note saying documents can be completed later before full activation
This should look polished but simple.

SCREEN 6 — Rancher signup step 4: payment preferences
Show simple payment setup options:
- instant payment
- payment at term
- configure later
Make this visually simple and reassuring.

SCREEN 7 — Account ready
Final screen confirming:
“Tu cuenta de Tropero está casi lista”
Show that the user can now continue to the rancher dashboard or create the first transport request later.
Include:
- Ir al panel
- Crear primer traslado
- Completar perfil después

Important design rules:
- Every onboarding step must include a visible option to skip or continue later
- The experience should feel smooth and low-friction
- Documents should be shown as frontend placeholders only
- Keep forms short and clean
- Use a progress indicator at the top
- Use modern card-based UI
- Use large buttons and clear hierarchy
- Make it mobile-friendly and desktop-friendly
- Prioritize the rancher experience only
- Keep the screens visually clean enough for a product demo, investor presentation, or developer handoff

Design this as a realistic SaaS/mobile-web onboarding flow for ranchers in Paraguay.
